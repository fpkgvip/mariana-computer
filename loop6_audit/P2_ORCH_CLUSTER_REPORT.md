## B-22: Atomic credit probe refund uses wrong RPC parameter names

### Root cause
`_atomic_probe_credits()` in `event_loop.py` calls `add_credits` with
`{"target_user_id": ..., "amount": 1}`. The live DB signature is
`public.add_credits(p_user_id uuid, p_credits integer)`. PostgREST raises
Postgres error 42883 (function not found) on every attempt. The `except
Exception` block catches it and retries three times; all three fail. The
user permanently loses 1 credit per probe call.

### Fix
**File:** `mariana/orchestrator/event_loop.py`
**Lines changed:** ~3353 (single line)

Before:
```python
await _rpc("add_credits", {"target_user_id": user_id, "amount": 1})
```
After:
```python
# B-22 fix: live DB signature is add_credits(p_user_id uuid, p_credits integer).
await _rpc("add_credits", {"p_user_id": user_id, "p_credits": 1})
```

## B-23: branch_manager imports non-existent `get_config` — config thresholds silently use hardcoded defaults

### Root cause
`_cfg_val()` in `branch_manager.py` imports `get_config` from `mariana.config`
which does not exist. The `ImportError` is caught by `except Exception` and the
function returns the hardcoded default. Six configuration thresholds
(`SCORE_KILL_THRESHOLD`, `SCORE_DEEPEN_THRESHOLD`, `SCORE_TRIBUNAL_THRESHOLD`,
`BUDGET_BRANCH_INITIAL`, `BUDGET_BRANCH_GRANT_SCORE7`, `BUDGET_BRANCH_GRANT_SCORE8`)
are never read from the environment.

### Fix
**File:** `mariana/orchestrator/branch_manager.py`
**Lines changed:** ~49-60 (function body), added `import os` at line ~18

Before:
```python
def _cfg_val(attr: str, default: float) -> float:
    try:
        from mariana.config import get_config  # noqa: PLC0415
        return float(getattr(get_config(), attr, default))
    except Exception:
        return default
```
After:
```python
def _cfg_val(attr: str, default: float) -> float:
    """...B-23 fix...
    Reads value directly from os.environ since AppConfig attribute names
    match env var names. Avoids calling load_config() which requires
    POSTGRES_DSN/POSTGRES_PASSWORD infrastructure credentials.
    """
    raw = os.environ.get(attr)
    if raw is None:
        return default
    try:
        return float(raw)
    except (ValueError, TypeError):
        return default
```

**Tradeoff:** Reading env vars directly (not via `load_config()`) keeps the
helper lightweight and testable in CI without a running Postgres. Since
`load_config()` reads the same env vars for these float attributes, the
behavior is identical. If AppConfig ever transforms/validates these values
before returning them, the helper would diverge — but currently all six
attributes are plain `_float(key, default)` reads in `load_config()`.

## B-24: Module-level `_metadata_lock` serializes all concurrent investigations

### Root cause
`event_loop.py` defined a single `_metadata_lock = asyncio.Lock()` at module
level. Both `handle_tribunal()` and `handle_skeptic()` acquire this lock to
perform a read-modify-write on their per-task `_tribunal_run_counter` /
`_skeptic_run_counter` metadata fields. In daemon mode with N concurrent
tasks, all N tasks contend on the same lock — task B stalls while task A
holds the lock (even though they operate on entirely different metadata dicts).

### Fix
**File:** `mariana/orchestrator/event_loop.py`
**Lines changed:**
- Lines ~99-125: removed `_metadata_lock = asyncio.Lock()`, added `_metadata_locks: dict[str, asyncio.Lock] = {}`, `_get_metadata_lock(task_id)`, and `_cleanup_metadata_lock(task_id)`
- Line ~2270: `async with _metadata_lock:` → `async with _get_metadata_lock(task.id):`
- Line ~2529: same replacement in `handle_skeptic`
- Line ~964: `_cleanup_metadata_lock(task.id)` added to `run()` finally block
- Added `from collections import defaultdict` import (unused directly but consistent with intent)

Before:
```python
_metadata_lock = asyncio.Lock()   # module-level, shared across all tasks
...
async with _metadata_lock:        # all concurrent tasks block here
    task.metadata["_tribunal_run_counter"] = ...
```
After:
```python
_metadata_locks: dict[str, asyncio.Lock] = {}

def _get_metadata_lock(task_id: str) -> asyncio.Lock:
    if task_id not in _metadata_locks:
        _metadata_locks[task_id] = asyncio.Lock()
    return _metadata_locks[task_id]

def _cleanup_metadata_lock(task_id: str) -> None:
    _metadata_locks.pop(task_id, None)
...
async with _get_metadata_lock(task.id):   # per-task, no cross-task contention
    task.metadata["_tribunal_run_counter"] = ...
```
